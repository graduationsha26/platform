Reduce the feature extraction window from 200 to 100 samples (stride 30→15) to halve the classification buffer, retrain the Random Forest classifier to produce a v3 model, update the inference service to load v3, and update the live test script to use probability-based prediction with a formatted confidence-scored terminal output. ## Phase 1: Design

### US1 — Reduce Window Memory & Retrain

**Flow**:
1. Edit `5_aggregate_and_extract.py`: `WINDOW_SIZE = 100`, `STRIDE = 15`
2. Execute `5_aggregate_and_extract.py` → regenerates `X_features.npy`, `y_labels.npy`
3. Edit `train_random_forest.py`: `MODEL_VERSION = 3`
4. Execute `train_random_forest.py` → produces `rf_model_v3.pkl`, `rf_model_v3_scaler.pkl`, `rf_model_v3.json`
5. Edit `inference/services.py`: update `model_map['rf']` and `metadata_map['rf']` to v3 paths
6. Verify: Django shell loads v3 model — confirms `window_size=100` in metadata, scaler not None

**Key invariant**: The scaler path is self-referential — the training script writes `metadata['scaler_file'] = 'rf_model_v3_scaler.pkl'` into the JSON. The inference service reads this path from the JSON. No explicit scaler path change needed in `services.py`.

### US2 — Confidence Scoring in Live Test

**Flow**:
1. Edit `live_glove_test.py`:
   - `WINDOW_SIZE = 200` → `100`
   - `--model` default → `rf_model_v3.pkl`
   - `--scaler` default → `rf_model_v3_scaler.pkl`
   - Replace `pred = model.predict(feature_scaled)[0]` with:
     ```python
     probs = model.predict_proba(feature_scaled)[0]  # shape (2,)
     pred = int(probs.argmax())
     confidence = probs[pred] * 100
     ```
   - Replace print statements with:
     ```python
     ts = datetime.now().strftime('%H:%M:%S.%f')[:12]
     if pred == 0:
         print(f'[{ts}] ✅ NORMAL (0) | Confidence: {confidence:.1f}%')
     else:
         print(f'[{ts}] ⚠️ TREMOR (1) | Confidence: {confidence:.1f}%')
     ```

**Note**: The `datetime` import must be verified present; the existing script uses a `ts` variable already computed from timestamps — confirm whether it uses `datetime.now()` or derives time from MQTT payload timestamp.

### No API Contracts Required

No new REST endpoints are created or modified for this feature. The existing `POST /api/inference/predict/` endpoint contract is unchanged. The inference service change is an internal implementation detail (model file path update only).
