# Data Model: On-Device Edge AI Tremor Classification

This feature is firmware-primary; its "data model" is the set of in-memory structures, the
serialized model format, and the canonical mapping shared across hosts. No database schema
changes (the backend remains authoritative for persisted records and its existing schema is
unchanged).

## 1. Canonical class mapping (single source of truth)

| Index | Symbol | Meaning | Drives suppression? |
|-------|--------|---------|---------------------|
| 0 | `NON_TREMOR` | Resting / no Parkinsonian tremor (Control) | No |
| 1 | `TREMOR` | Active Parkinsonian tremor | **Yes** |
| 2 | `VOLUNTARY` | Intentional movement (confound) | No |

- Represented on-device as `enum class TremorClass : uint8_t { NON_TREMOR = 0, TREMOR = 1, VOLUNTARY = 2 };`
- Mirrors `backend/ml_models/train.py` `CLASS_NAMES`/`GROUPS` and the model metadata `classes`.
- **Validation rule**: a compile-time/static assertion (and a parity test) confirms the device
  enum values equal the training label integers; the suppression gate keys off `TREMOR` only.

## 2. Sample & window structures (firmware)

### `ImuSample`
Latest calibrated 6-axis reading already produced by SensorTask.
- `float ax, ay, az;`   // m/s²
- `float gx, gy, gz;`   // °/s
- (Source: existing `FusedReading`; the classifier consumes the 6 raw axes, not roll/pitch.)

### `RingBuffer` (sliding analysis window)
- Capacity: `WINDOW_SIZE = 128` samples per axis (1.28 s @ 100 Hz).
- Layout: 6 fixed-length float ring buffers (or one `float[6][128]`), head index, fill count.
- **Write**: SensorTask pushes one `ImuSample` per 10 ms (non-blocking, overwrites oldest).
- **Read**: ClassificationTask snapshots the buffer every `HOP = 10` samples (~100 ms).
- **State**: `ready` once `fill_count >= WINDOW_SIZE` (warm-up gate, FR-011).
- **Validation rule**: a window with any non-finite or out-of-range value (|accel|, |gyro|
  beyond sensor range) is marked invalid → decision withheld for that cycle.

### `FeatureVector`
- `float feat[66];` — axis-major, order **identical** to `get_feature_names_66()`:
  per axis `[mean, std, median, q1, q3, min, max, peak1_freq, peak1_amp, peak2_freq, peak2_amp]`,
  axes `aX, aY, aZ, gX, gY, gZ`.
- **Validation rule**: index *i* of `feat` must equal the Python reference feature *i* within
  the numerical tolerance defined in research.md (SC-006).

## 3. Transpiled model format (`tremor_model.h/.cpp`, generated)

Generated by `backend/ml_models/export_to_c.py` from `lgbm_tremor_model.pkl`. Flat-array,
read-only (flash) representation of the boosted ensemble.

### Per-node arrays (one set, all trees concatenated)
- `int16_t  feature[]`   — split feature index, or `-1` for a leaf
- `float    threshold[]` — split threshold (numerical `<=` comparison; see research.md)
- `int32_t  left[]`      — index of left child (condition true), or leaf payload sentinel
- `int32_t  right[]`     — index of right child (condition false)
- `uint8_t  default_left[]` — NaN/default direction (defensive; device inputs are finite)
- `float    leaf_value[]` — raw additive score for leaf nodes

### Ensemble metadata (constants)
- `NUM_CLASS = 3`, `NUM_ITER` (= n_estimators), tree→class mapping (round-robin: tree *t*
  contributes to class `t % NUM_CLASS`), per-tree root offsets, `init_score` per class if any.
- **Prediction**: sum leaf raw scores per class across that class's trees → `softmax` →
  `argmax` = predicted class; softmax outputs = `proba[3]`.

### Validation rules
- The exporter MUST refuse to emit categorical splits (the pipeline uses only numerical
  features) and assert the comparison operator matches LightGBM's default.
- Regenerating from the same `.pkl` MUST produce byte-identical arrays (SC-008 reproducibility).
- Array sizes recorded as constants; build fails if interpreter and arrays disagree.

## 4. On-device decision & gate state

### `Decision`
- `TremorClass cls;`
- `float proba[3];`
- `uint32_t t_decision_us;`
- `bool valid;` (false during warm-up / invalid window)

### `GateState` (smoothed suppression gate — FR-003b)
- `enum class Gate { DISENGAGED, ENGAGING, ENGAGED, DISENGAGING };`
- Sliding vote buffer of the last `N_VOTE` decisions (e.g., last 5 = ~500 ms).
- Engage when a sustained majority of recent windows are `TREMOR` (hysteresis threshold
  `ENGAGE_VOTES`); disengage only after a sustained majority of non-`TREMOR`
  (`DISENGAGE_VOTES`, with `DISENGAGE_VOTES` > margin so it does not flip back instantly).
- Optional ramp on engage/disengage so actuator authority changes smoothly, not as a step.
- **Validation rule (SC-009)**: alternating borderline decisions MUST NOT change `Gate`
  more than once per defined minimum dwell time → no per-cycle actuator chatter.
- **Default-safe rule**: on `Decision.valid == false` (warm-up/invalid), the gate holds its
  current state if `ENGAGED`→biases toward a controlled `DISENGAGING` after a timeout; it
  never engages suppression from an invalid/unknown state.

## 5. Relationships / data flow

```
SensorTask(100Hz) --ImuSample--> RingBuffer
RingBuffer --(every 100ms snapshot)--> ClassificationTask(Core0)
ClassificationTask: window -> edge_features(66) -> tremor_model interpreter -> softmax -> Decision
Decision --> GateState(vote/hysteresis) --> ControlTask(200Hz) engages/disengages PID
(optional) Decision --> MqttTask --> existing telemetry channel --> backend (records, authoritative)
```

## 6. Cross-host invariants (must not drift)

- Feature definition (order, window length, FFT length, filter coefficients, sample rate) is
  identical in `features_lgbm.py` (reference), `export_to_c.py`-era assumptions, and
  `edge_features.cpp`.
- Class mapping identical in training, backend serving, device enum, and gate.
- The C model arrays are derived purely from the trained `.pkl`; the `.pkl` is the only source
  of truth for weights.
