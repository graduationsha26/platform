# Implementation Plan: Update User Model Roles

**Branch**: `012-update-user-roles` | **Date**: 2026-02-18 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/012-update-user-roles/spec.md`

## Summary

Remove the `patient` role from the `CustomUser` model's valid role choices and add `admin` as a valid role. Set `doctor` as the default role. Update the registration serializer's `validate_role` method to enforce the new valid set. Create a Django migration to apply the field change to the database. Remove the now-dead `is_patient()` method and add an `is_admin()` helper. This is a narrowly scoped change to the `authentication` app only.

## Implementation Design

### 1. Model Change (`authentication/models.py`)

```python
ROLE_CHOICES = [
    ('doctor', 'Doctor'),
    ('admin', 'Admin'),
]

role = models.CharField(
    max_length=10,
    choices=ROLE_CHOICES,
    default='doctor',
    help_text='User role: doctor or admin'
)

def is_doctor(self):
    return self.role == 'doctor'

def is_admin(self):           # NEW
    return self.role == 'admin'

# is_patient() — REMOVED
```

### 2. Migration (`authentication/migrations/0002_alter_customuser_role.py`)

Django `AlterField` on `role`:
- Updated `choices` list
- Added `default='doctor'`
- Updated `help_text`

### 3. Serializer Change (`authentication/serializers.py`)

```python
def validate_role(self, value):
    if value not in ['doctor', 'admin']:
        raise serializers.ValidationError("Role must be either 'doctor' or 'admin'.")
    return value
```

### 4. Constitution Update (`.specify/memory/constitution.md`)

Principle IV, "Non-negotiable constraints" bullet: update `Two roles: 'patient' and 'doctor'` → `Two roles: 'doctor' and 'admin'`.
