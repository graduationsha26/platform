# Implementation Plan: Reports Page & PDF Download

**Branch**: `035-reports-pdf` | **Date**: 2026-02-21 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/035-reports-pdf/spec.md`

## Summary

A doctor-facing Reports page at `/doctor/patients/:id/reports` that lets doctors select a date range, preview four aggregated tremor metrics (avg amplitude, max amplitude, dominant frequency, tremor reduction %), and download a formatted PDF report — all via two existing backend API endpoints (`GET /api/analytics/stats/` and `POST /api/analytics/reports/`).

**This is a pure frontend feature.** No new backend models, migrations, or endpoints are required.

## Technical Context

**Frontend Stack**: React 18+ + Vite + Tailwind CSS
**Database**: Supabase PostgreSQL (remote) — no changes

**Testing**: Not requested

**Performance Goals**: Stats preview loads within 3 seconds; PDF download starts within 10 seconds
**Constraints**: Local development only; no Docker/CI/CD; no new npm dependencies
**Scale/Scope**: Single-patient scoped; date range up to 365 days; ~30 days default

## StatsSummary Aggregation (Client-Side)

Computed from the `results` array returned by `GET /api/analytics/stats/`:

```js
const computeSummary = (results) => ({
  hasData: results.length > 0,
  sessionCount: results.reduce((s, r) => s + r.session_count, 0),
  avgAmplitude: results.reduce((s, r) => s + r.avg_amplitude, 0) / results.length,
  maxAmplitude: Math.max(...results.map(r => r.avg_amplitude)),
  dominantFrequency: results.reduce((s, r) => s + r.dominant_frequency, 0) / results.length,
  tremorReductionPct: (() => {
    const valid = results.filter(r => r.tremor_reduction_pct != null);
    return valid.length > 0
      ? valid.reduce((s, r) => s + r.tremor_reduction_pct, 0) / valid.length
      : null;
  })(),
});
```

## PDF Download Implementation

```js
const downloadPDF = async () => {
  setDownloadLoading(true);
  setDownloadError(null);
  try {
    const blob = await downloadPatientReport(patientId, dateRange.startDate, dateRange.endDate);
    const url = URL.createObjectURL(new Blob([blob], { type: 'application/pdf' }));
    const a = document.createElement('a');
    a.href = url;
    a.download = `report_patient${patientId}_${dateRange.startDate}_${dateRange.endDate}.pdf`;
    a.click();
    URL.revokeObjectURL(url);
  } catch (err) {
    const code = err.response?.data?.code;
    if (code === 'NO_DATA_FOR_REPORT') {
      setDownloadError('No data available for this period — try a different date range.');
    } else if (code === 'PDF_SIZE_LIMIT_EXCEEDED') {
      setDownloadError('Report too large — try a smaller date range (max ~90 days recommended).');
    } else {
      setDownloadError('Report generation failed — please try again.');
    }
  } finally {
    setDownloadLoading(false);
  }
};
```

**Note**: When `downloadPatientReport` uses `responseType: 'blob'`, Axios wraps errors differently — the error blob must be read as text to extract the `code` field. Handle this in the service or hook with a `FileReader` or `blob.text()`.
