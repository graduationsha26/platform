# Implementation Plan: Update Patient Model

**Branch**: `014-update-patient-model` | **Date**: 2026-02-18 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/014-update-patient-model/spec.md`

## Summary

Remove the `Patient.user` OneToOneField (which linked patient records to user authentication accounts). Since E-1.1 removed 'patient' from `ROLE_CHOICES`, this link is permanently dead. The required clinical fields (`full_name`, `contact_phone`, `medical_notes`) already exist under their current names — no new fields needed. The doctor-patient assignment relationship (via `DoctorPatientAssignment` junction) is preserved unchanged. The change requires one DROP COLUMN migration and cleanup of 4 code locations that reference the now-removed field.

## Implementation Design

### Change 1: Remove `Patient.user` Field

**File**: `backend/patients/models.py`

**Before**:
```python
user = models.OneToOneField(
    'authentication.CustomUser',
    on_delete=models.SET_NULL,
    null=True,
    blank=True,
    related_name='patient_profile'
)
```

**After**: Field entirely removed.

### Change 2: Migration

**File**: `backend/patients/migrations/0002_remove_patient_user.py`

Generated by `python manage.py makemigrations patients` after removing the field. The migration will contain a `RemoveField` operation that drops the `user_id` column from the `patients` table.
